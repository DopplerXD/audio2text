可以把方案大幅收缩为一个**单进程、本地运行的音频转写工具**：

```text
浏览器页面（Gradio）
        ↓
Python 转写脚本
        ↓
MLX Whisper
        ↓
结构化分段结果
        ↓
TXT / PDF / SRT / VTT / JSON
```

不需要前后端分离，不需要数据库，也不需要 Agent、HTTP 服务或任务调度。

## 一、推荐技术方案

| 模块       | 选择                                   | 原因                                                         |
| ---------- | -------------------------------------- | ------------------------------------------------------------ |
| 本地页面   | **Gradio**                             | 几十行 Python 就能实现文件上传、参数选择、文本展示和文件下载 |
| 语音识别   | **mlx-whisper**                        | 专门面向 Apple Silicon 的 MLX 实现，适合你的 MacBook Air M5 32GB |
| 默认模型   | `mlx-community/whisper-large-v3-turbo` | 多语言，中文、英文、粤语等混合录音都能处理                   |
| 音视频处理 | **FFmpeg**                             | 统一处理 MP3、M4A、WAV、MP4、MOV 等格式                      |
| PDF 导出   | **ReportLab**                          | Python 本地直接生成 PDF                                      |
| 字幕导出   | 自己生成 SRT/VTT                       | 根据 Whisper 返回的分段时间戳生成                            |
| 项目管理   | Python venv 或 uv                      | 保持依赖简单                                                 |

MLX Whisper 官方支持直接处理音频文件、指定模型，并返回分段结果和词级时间戳；CLI 本身也能通过 `-f` 选择输出格式。([GitHub](https://github.com/ml-explore/mlx-examples/blob/main/whisper/README.md))

`whisper-large-v3-turbo` 的 MLX 转换模型约 1.61GB，你的 32GB 统一内存足够使用，不需要优先选择 tiny 或 base。([Hugging Face](https://huggingface.co/mlx-community/whisper-large-v3-turbo?utm_source=chatgpt.com))

## 二、页面设计

页面保持成左右两栏即可。

### 左侧：上传与配置

- 上传音频或视频
- 音频播放器
- 语言：
    - 自动识别
    - 中文
    - 英文
    - 日语
    - 粤语
- 模型：
    - 快速模式
    - 标准模式
    - 高质量模式
- 是否生成词级时间戳
- “开始识别”按钮
- 识别进度和耗时

Gradio 的 File 组件支持限制音频、视频或具体扩展名，也支持单文件和多文件上传；下载按钮可以直接接收 Python 生成的文件路径。([gradio.app](https://www.gradio.app/docs/gradio/file))

### 右侧：结果编辑与导出

不要只放一个大文本框，建议分成两个标签页：

**完整文本**

```text
大家好，今天我们讨论一下本地语音识别工具……
```

用于编辑和导出：

- TXT
- Markdown
- PDF

**字幕分段**

| 开始时间     | 结束时间     | 内容                       |
| ------------ | ------------ | -------------------------- |
| 00:00:00.000 | 00:00:04.320 | 大家好，今天我们讨论一下…… |
| 00:00:04.320 | 00:00:08.760 | 首先介绍本地语音识别……     |

每个分段可以修改，用于导出：

- SRT
- VTT
- JSON

最好以“字幕分段表”为核心数据。用户修改分段内容后，再重新拼接出完整文本，这样 TXT、PDF、SRT 的内容能够保持一致。

## 三、项目结构

```text
local-transcriber/
├── app.py                 # Gradio 页面和事件绑定
├── transcriber.py         # MLX Whisper 调用
├── audio_utils.py         # FFmpeg 转换、音频切分
├── exporters.py           # TXT/PDF/SRT/VTT/JSON 导出
├── models.py              # Segment、TranscriptionResult 数据结构
├── requirements.txt
├── temp/                  # 临时音频
└── outputs/               # 转写结果
```

不要拆成 Vue/React 前端和 FastAPI 后端。对这种个人本地工具而言，Gradio 页面直接调用 Python 函数更简单。

内部统一保存成一个结构：

```python
{
    "filename": "meeting.m4a",
    "language": "zh",
    "duration": 1832.5,
    "text": "完整识别文本……",
    "segments": [
        {
            "id": 0,
            "start": 0.0,
            "end": 4.32,
            "text": "大家好，今天我们讨论一下……"
        }
    ]
}
```

之后所有导出器只读取这个结构，避免 TXT、PDF、字幕各写一套识别逻辑。

## 四、识别与导出流程

```text
用户上传文件
    ↓
检查扩展名、文件大小
    ↓
FFmpeg 转成 16kHz 单声道 WAV
    ↓
mlx-whisper 执行识别
    ↓
提取 text 和 segments
    ↓
页面展示并允许编辑
    ↓
用户选择导出格式
    ↓
生成单个文件或 ZIP 压缩包
```

核心识别代码可以保持很简单：

```python
import mlx_whisper

MODEL_NAME = "mlx-community/whisper-large-v3-turbo"


def transcribe_audio(
    audio_path: str,
    language: str | None = None,
) -> dict:
    options = {
        "path_or_hf_repo": MODEL_NAME,
        "word_timestamps": True,
    }

    if language and language != "auto":
        options["language"] = language

    result = mlx_whisper.transcribe(
        audio_path,
        **options,
    )

    return {
        "text": result["text"].strip(),
        "language": result.get("language"),
        "segments": [
            {
                "id": index,
                "start": segment["start"],
                "end": segment["end"],
                "text": segment["text"].strip(),
                "words": segment.get("words", []),
            }
            for index, segment in enumerate(result["segments"])
        ],
    }
```

MLX Whisper 官方 Python API就是返回包含 `text` 和 `segments` 的结果，并支持 `word_timestamps=True`。([GitHub](https://github.com/ml-explore/mlx-examples/blob/main/whisper/README.md))

### 各格式内容

**TXT**

```text
大家好，今天我们讨论一下本地语音识别工具。
首先介绍一下项目的基本结构。
```

**带时间戳 TXT**

```text
[00:00:00 - 00:00:04] 大家好，今天我们讨论一下本地语音识别工具。
[00:00:04 - 00:00:09] 首先介绍一下项目的基本结构。
```

**SRT**

```srt
1
00:00:00,000 --> 00:00:04,320
大家好，今天我们讨论一下本地语音识别工具。

2
00:00:04,320 --> 00:00:08,760
首先介绍一下项目的基本结构。
```

**PDF**

建议包含：

- 文件名
- 音频时长
- 识别语言
- 生成时间
- 完整文本
- 可选的分段时间戳

生成中文 PDF 时，需要注册 macOS 自带中文字体，例如苹方字体，否则 ReportLab 默认字体可能无法正常显示中文。

## 五、模型和版本规划

### 第一版只使用 MLX Whisper

建议默认：

```text
mlx-community/whisper-large-v3-turbo
```

它支持多语言、语音翻译和语言识别，而 MLX Whisper 可以直接使用 Hugging Face 上预转换的模型。([GitHub](https://github.com/openai/whisper))

页面上不必一开始就提供十几个模型，只提供三个用户概念：

| 页面选项 | 实际模型         |
| -------- | ---------------- |
| 快速     | small 或量化模型 |
| 标准     | medium           |
| 高质量   | large-v3-turbo   |

你的设备可以默认选择“高质量”。

### 后续再增加中文专用模式

等第一版完成后，可以再增加一个识别引擎选择：

```text
通用模式：MLX Whisper
中文模式：FunASR Paraformer
```

FunASR 官方目前把 SenseVoice-Small 推荐为通用起点，把 Paraformer 推荐给以普通话为主的生产识别场景，并支持 VAD、标点和说话人区分。([GitHub](https://github.com/modelscope/FunASR/blob/main/docs/model_selection.md))

但第一版不要同时接入两个框架。FunASR 依赖 PyTorch，Mac 上的部署和性能路径没有 MLX Whisper 直接，容易让项目再次变复杂。

## 最终建议的第一版范围

只实现以下内容：

- 上传单个音频或视频
- 本地 MLX Whisper 识别
- 自动语言识别
- 展示完整文本和字幕分段
- 手动修改识别结果
- 导出 TXT、PDF、SRT、VTT、JSON
- 一键打包所有格式为 ZIP
- 所有文件仅在本地处理

明确暂不实现：

- 实时麦克风转写
- 多人说话人识别
- 批量任务队列
- 用户系统和历史数据库
- Agent、LLM 总结、会议纪要
- 前后端分离
- 桌面应用打包

基础环境安装可以收敛到：

```bash
brew install ffmpeg

python3 -m venv .venv
source .venv/bin/activate

pip install mlx-whisper gradio reportlab

python app.py
```

这个方案的代码量预计可以控制在 **300～500 行 Python** 左右，第一版的核心就是 `app.py + transcriber.py + exporters.py` 三个文件。
